步驟1：開立變更單

執行者： 提出團隊（推動／規劃）
目標： 啟動版更流程，提供完整的更新需求資訊，以便開發團隊及相關單位進行準備。

  • • 由提出團隊於 ASE 系統建立變更申請單。
  • • 依以下規範完整填寫更新申請內容：標題格式：更新日期（MM/DD）hh:mm AP／DB／HW 系統名稱。
  • • 1. 主機資訊（IP、Host Name）
  • • 2. 影響範圍（國家、事業群、區域、工廠）
  • • 3. 上版內容（必須附上申請單連結與 ASE 單號，並上傳測試報告文件）
  • • 4. 特殊事項（例如：需 DBA 於更新前備份、預計停機時間、其他注意事項）
  • • 5. 轉組態異動單： Y / N（若需修改 CMDB，請選 Y）
  • • 將申請單轉派給開發團隊準備上版資料。

步驟2：準備上版資料

執行者： 開發團隊
目標： 準備完整且經測試驗證的更新版本，確保品質，供上版團隊執行。

  • • 開發團隊接收提出團隊的變更申請單。
  • • 檢查申請內容，如有不清楚之處，請於「討論區」回覆並通知提出團隊。
  • • 準備上版資料，並進行測試（測試對象由開發團隊與提出團隊自行確認）。
  • • 測試完成後，於變更申請單中附上測試報告檔案。
  • • 開發團隊於申請單上註記：上版資料準備完成，提供儲存位置與執行順序，供上版團隊了解如何設定上版參數。
  • • 準備完成後：將變更申請單轉派上版團隊以準備系統版更，同時通知提出團隊進行送審變更申請單。

步驟3：系統版更執行

執行者： 上版團隊（DBA）
目標： 依開發團隊提供的上版資料進行系統版更，並完整記錄結果。

  • • 確認 IT 部門主管之核准狀態。
  • • 依核准排程執行系統版更。
  • • 於變更單中記錄版更結果。
  • • 將申請單轉派回提出團隊進行確認與驗收。

步驟4：版更後結果確認

執行者： 提出團隊
目標： 確保系統版更符合需求，資料穩定。

  • • 檢查版更後之系統運作狀態。
  • • 於變更申請單上記錄結果（正常／異常）。
  • • 若發現異常，請於「討論區」詳細說明問題，並將單據退回給開發團隊，流程回到步驟 2重新執行。
  • • 若無異常但需修改 CMDB，則轉為組態異動單並派給開發團隊執行及結案。
  • • 所有確認完成後，結案變更申請單，版更流程結束。

重要提醒

  • • 變更申請單須經 IT 部門主管核准後方可生效。
  • • 變更申請單完成檢查並完整記錄進度後即可結案。
